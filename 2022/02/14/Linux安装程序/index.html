介绍Linux上安装程序的几种方法和规范

原理

Linux可识别的包

源代码包

  • 优点：开源：如果有足够能力，可以修改源代码；可以自由选择所需的功能；软件是编译安装，所以更加适合自己的系统，更加稳定也效率高；卸载方便：直接删除文件夹。
  • 缺点：安装过程步骤较多，尤其是安装较大的软件集合（如：LAMP环境搭建），容易出现拼写错误；编译时间较长，安装比二进制安装时间长；因为是编译安装，安装过程中一旦报错新手很难解决。

.rpm包/.deb包(二进制包)

  • .rpm用rpm: Redhat Package Manager

  • .deb用dpkg: Debian Package

由于RPM出现得更早，所以在各种版本的Linux都常见到。

而dpkg则只出现在Debian Linux中，其它Linux版本一般都没有。

  • 二进制包：与源代码区别就是源代码包经过编译成为机器能直接识别的二进制包
  • 优点：包管理系统简单，只需要几个命令就可以实现包的安装、升级、查询和卸载；安装速度比源码包快。
  • 缺点：经过编译，不可以看到源代码；功能选择不如源代包灵活；依赖性。

RPM默认安装路径

下载包

  1. ftp文件传输软件
  2. wget直接从网站下载

安装

  • rpm包的安装步骤

    1. 找到包并下载到本机某目录
    2. cd .rpm包所在的目录
    3. rpm -ivh <包名>

  • deb包的安装步骤

    1. 找到包并下载到本机某目录
    2. cd .deb包所在的目录
    3. dpkg -i <包名>

  • 源代码包的安装步骤：

    1. 下载

    2. cd 包所在目录

    3. 解压：

    4. 进入包cd soft

    5. ./configure 检查配置

    6. make编译

    7. make install安装

    8. make clean清楚编译和配置过程中的临时文件

  • 使用 yum / apt-get 直接安装

    • yum是基于rpm开发，自动处理依赖关系，适用于Fedora/CentOS/RedHat

    • apt-get，类yum，适用于Ubuntu/Debian
